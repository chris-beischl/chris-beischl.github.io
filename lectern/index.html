The cycle

One round, start to finish — every time.

Each book your club reads runs through the same five phases. Lectern drives them automatically, and an admin can jump in or skip ahead whenever the plan changes.

Getting started

Set up your club in a few minutes.

Four short steps and you're reading. Everything is configured per server with slash commands — no dashboards, no accounts. Admin steps need the Manage Server permission.

1

Add Lectern & run setup

Invite the bot to your server, then run /setup to pick the announcement channel and language. That activates your club — everything else has sensible defaults you can tune later.

/setup
2

Set the meeting rhythm

/schedule sets how often you meet (e.g. the 2nd Saturday each month) and the minimum reading time in days. Lectern uses this to plan every cycle backwards from the meeting.

/schedule
3

Tune the club (optional)

Run /config with no options for an interactive panel — proposals & votes per person, quorums, phase windows, tie-breaks, the scheduling mode, and the autopilot (let cycles start themselves on the schedule).

/config
4

Start reading

Open proposals with /startcycle — or, if the book is already decided, jump straight ahead with /newcycle. With the autopilot on, you don't even need this: Lectern begins each round on its own.

/startcycle /newcycle

The settings worth knowing

Proposals / votes per person — how many books each member may propose and how many votes they get.
Min. readers / attendees — the quorum a round needs to proceed to scheduling and to lock a date.
Phase windows & reading days — how long proposing, voting, sign-up and scheduling stay open, and the minimum reading time.
Tie-breaks — settle a level vote by runoff, random pick, or an admin's choice.
Scheduling mode — take the next fixed slot, or poll readers for the dates that suit them.
Autopilot — start cycles automatically on the schedule, with skip or compress when a start runs late.

Privacy & data

Your club's data, treated with care.

Lectern is built for European data protection from the ground up — minimal data, no message snooping, and tools for members to see or erase what's theirs.

Data minimisation

Only what's needed: a pseudonymous user ID, your display name, and the books, votes and RSVPs you submit. No emails, no message content.

Hosted in the EU

Data is stored on EU infrastructure and backups are treated as personal data. Retention is documented and limited.

See & erase, anytime

/my-data shows what's stored; /forget-me anonymises your votes and deletes the rest.

Leave, and it's gone

Remove Lectern from a server and the club's data is purged after a short, documented retention window.
